# 📺 YouTube 記事変換ツール (YouTube to Article Tool)

このツールは、YouTube動画の内容を迅速に深く構造化された記事に変換するAI搭載ツールです。`yt-dlp`で字幕を抽出し、Google Gemini AIを用いて論理的に再構成することで、口語的な動画内容を高品質な書き言葉の知識ベースに変換します。手動でメモを取る必要はもうありません — 教学動画、インタビュー、講演などをワンクリックで再利用可能なMarkdown記事に変換できます。

---

## ✨ 主な機能

### ⚡ 並列変換
複数のYouTubeリンクを一度に入力でき、`asyncio.gather`による非同期I/Oで複数の動画を同時に処理します。大量のバッチ変換でも効率的に動作します。

### 🧠 深い構造化
単なる要約ではなく、AIが口語的な内容から3〜5個の核心テーマを抽出し、**テーマブロック**（Topic Blocks）形式で内容を再構成します。具体的な事例、データ、金言、ストーリーを保持し、動画を見なくても完全に理解できる記事を生成します。

### 🎯 スマート字幕マッチング
以下の優先順位で利用可能な最適な字幕を自動選択します：

`["zh", "zh-TW", "zh-HK", "zh-CN", "zh-Hans", "zh-Hant", "en", "ja", "ko"]`

- 優先的に中国語（繁体字/簡体字）を選択
- 次に英語、日本語、韓国語を順にフォールバック
- **VTT / SRT / JSON の3形式に対応**。優先言語リストで一致する字幕がない場合、自動的に `--sub-langs all` で再試行
- YouTube自動生成字幕（Auto-generated captions）にも対応

### 🌍 多言語サポート
- **UI言語**：4種類のインターフェース切り替え対応 — 日本語、English、繁體中文、简体中文
- **出力言語**：7種類の出力言語に対応（日本語、英語、中国語繁体字/簡体字、韓国語など）

### 📜 履歴管理
- ローカル保存先：`data/history.json`
- 自動重複排除：同一URLは最新の1件のみ保持
- 上限50件、超過時は最古のものから自動削除
- 各履歴の削除に対応（DELETE /history endpoint）

### 📥 Markdown 書き出し
- 明確な見出し階層と太字による強調
- 元動画の情報ブロック（タイトル、チャンネル名など）
- ワンクリックでクリップボードにコピー（Toast通知で確認）
- ワンクリックで `.md` ファイルをダウンロード
- Obsidian、Notionなどのナレッジ管理ツールに完全対応

## 🚀 クイックスタート

### 1️⃣ 環境準備

システムに以下の依存関係がインストールされていることを確認してください：

**Python 3.9+**
```bash
python3 --version
```

**yt-dlp**（コア字幕抽出ツール）
```bash
pip install yt-dlp
```

**必要な Python ライブラリ**
```bash
pip install fastapi uvicorn httpx python-multipart
```

または、プロジェクトの `requirements.txt` を使用：
```bash
pip install -r requirements.txt
```

### 2️⃣ API Key 設定

本ツールには **Google Gemini API Key** が必要です。設定の優先順位は以下の通りです：

| 優先順位 | 方法 | 説明 |
|---------|------|------|
| 1（最優先） | フロントエンドフォーム入力 | Web UIで直接入力。ブラウザの `localStorage` のみに保存 |
| 2（予備） | 環境変数 | `GEMINI_API_KEY` を設定しておけば毎回の入力を省略可能 |

```bash
export GEMINI_API_KEY="***"
```

どちらも設定されていない場合は、ツールはエラーを返し、入力を促します。

> **セキュリティ**：API Keyはサーバー側に保存されません。フロントエンドの `localStorage` に一時的にキャッシュされるのみです。

### 3️⃣ 起動

```bash
bash start.sh
```

起動後、ブラウザで開く：
```
http://127.0.0.1:8080
```

`start.sh` は以下の処理を自動実行します：
1. プロジェクトディレクトリに移動
2. `requirements.txt` の依存関係をインストール
3. `uvicorn` でFastAPI開発サーバーを起動（`--reload` ホットリロード付き）

### 4️⃣ 使用方法

変換プロセスは簡単4ステップ：

| ステップ | 操作 | 説明 |
|---------|------|------|
| 1 | API Keyを入力 | Gemini API Keyを入力（環境変数設定済みの場合はスキップ可） |
| 2 | 出力言語を選択 | 記事の出力言語を選択 |
| 3 | YouTubeリンクを貼り付け | 1行に1つ、カンマやスペース区切りも自動解析 |
| 4 | 変換開始 | 「変換開始」をクリック、または `Ctrl/Cmd + Enter` ショートカット |

変換完了後、結果はタブ（Tabs）形式で表示され、複数記事を簡単に切り替えて閲覧できます。

### v3.4 (2026-05-01) — メタデータ抽出の弾力性強化

**🐛 Bug 修正**
- **BrokenPipeError 自動リトライ**：`--dump-json` を `get_video_meta()` 関数に分離、3 回の自動リトライ機構（1s/2s/3s 間隔）を実装。長尺動画（118KB+ メタデータ）で発生する断続的なパイプ切断が自動復旧する。
- **一時ファイルの確実なクリーンアップ**：`task_temp` と `meta_file` の削除を `finally` ブロックに移動。例外発生時も一時ファイルが確実に削除される。
- **メモリ効率最適化**：`stdout.decode()` → ファイル書込 → `json.load()` の冗長フローを `json.loads()` に置き換え、I/O とデコードを削減。
- **統一 `run_with_retry()`**：すべての `subprocess.run` 呼出しを `run_with_retry()` ラッパーに統合。自動 BrokenPipeError リトライ（3 回、線形バックオフ）。従来は `--dump-json` のみ保護されていたが、字幕ダウンロード（`--sub-langs all` 含む）もパイプ切断時に自動復旧する。
- **最終リトライの無駄な sleep を除去**：最終失敗後の 3 秒待機を廃止。

### v3.3 (2026-05-01) — 字幕抽出の信頼性強化

**🐛 Bug 修正**
- **BrokenPipeError 修正**：`subprocess.run(stdout=open(...))` を `capture_output=True` に変更。yt-dlp が大量の JSON メタデータ（60万+ 文字）を出力する際の `[Errno 32] Broken pipe` を防止。
- **429 による一括失敗を修正**：字幕ダウンロードコマンドに `--ignore-errors` を追加。従来は単一の言語が HTTP 429 を返すと yt-dlp が**全体を中断**（return code 1、ディスクにファイルが 0 個）していた。現在は成功した言語は個別の失敗に影響されずに処理を継続。
- **言語優先順位の調整**：`en` を `common_langs` リストの先頭に移動。ほとんどの動画の原語は英語であり、`en` を先頭にすることで自動翻訳字幕の 429 トリガーを回避。
- **メタデータ取得のエラーハンドリング**：`--dump-json` サブプロセスが失敗時にキャッチされない例外ではなく、明確なエラーメッセージを返すように改善。

## ⚠️ 注意事項

### 🔑 API Key のセキュリティ
- **API Keyはサーバーに保存されません**
- Keyはブラウザの `localStorage` にのみ一時的に保存され、リクエスト時にフロントエンドからバックエンドを経由してGoogle Geminiに送信されます
- ブラウザタブを閉じるかサイトデータをクリアすると、Keyは削除されます

### 🎬 字幕への依存
- 本ツールは**動画の字幕（自動生成字幕を含む）に完全に依存**します
- **VTT / SRT / JSON** の3種類の字幕形式に対応し、各形式から自動的にテキストを抽出・クリーンアップします
- 字幕が全くない動画（音楽のみ、無声動画、字幕機能が無効な動画など）は変換できません
- 事前にYouTubeプレーヤーで「CC」字幕が有効にできるか確認することを推奨します

### ⏱️ 長時間動画
- 1時間以上の動画は字幕が長くなり、AIの処理がタイムアウトする場合があります
- ツールには**Map-Reduce分割処理**が内蔵されており、字幕を分割して処理してから統合することで、コンテキストウィンドウの制限を回避します
- 長時間動画で変換に失敗した場合は、動画を短く分割するか、分割処理の利用を検討してください

### 🚦 429 レート制限（字幕抽出）

429 は YouTube-to-Article で最も一般的なエラー原因です。その動作メカニズムを理解することで迅速なトラブルシューティングが可能になります：

- **問題**：短時間に大量に YouTube にアクセスすると **429 Too Many Requests** 制限が発生する場合があります
- **yt-dlp のデフォルト動作**：リクエスト内の**どれか一つの言語**が 429 を返すと、yt-dlp は全バッチを**中断**（return code 1、ディスクにファイルが 0 個）。これは yt-dlp の設計であり、バグではありません
- **ツールの対応**：字幕ダウンロードコマンドに `--ignore-errors` を追加（v3.3+）。成功した言語は個別の失敗に影響されずに処理を継続します
- **`[Errno 32] Broken pipe` の実態**：`yt-dlp --dump-json` が大量のメタデータを出力する際（長尺動画で 118KB+）、パイプがデータ量過多で切断されることがある。v3.4 では専用 `get_video_meta()` 関数を導入し、3 回の自動リトライでパイプ切断が自動復旧する。
- **続く場合**：数分待ってから再試行するか、同時実行するバッチ数を減らしてください

- 削除操作：履歴パネルの削除ボタンをクリックして即時削除
